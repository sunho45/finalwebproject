# 공부계획 기록 사이트 발표문

안녕하세요. 제가 만든 프로젝트는 공부계획을 기록하고 관리할 수 있는 웹사이트입니다.

이 사이트는 하루에 해야 할 공부나 과제를 간단하게 등록하고, 완료 여부를 체크하면서 전체 진행 상황을 확인할 수 있도록 만들었습니다. 사용자는 할 일, 과목, 날짜, 예상 소요 분을 입력할 수 있고, 등록된 계획은 목록에서 바로 확인할 수 있습니다. 완료한 항목은 체크해서 구분할 수 있으며, 전체 계획 수, 완료한 계획 수, 누적 공부 분도 함께 표시됩니다.

이 프로젝트를 만들 때 가장 중요하게 생각한 부분은 단순한 할 일 목록이 아니라, 실제로 공부 계획을 계속 기록할 수 있는 구조를 만드는 것이었습니다. 그래서 화면에서만 데이터가 사라지는 방식이 아니라, 서버와 데이터베이스를 연결해서 계획 데이터가 PostgreSQL에 저장되도록 구현했습니다. 사용자가 계획을 추가하거나 삭제하고, 완료 상태를 변경하면 Express 서버의 API를 통해 데이터베이스에 반영됩니다.

프론트엔드는 React와 Vite를 사용했습니다. React의 상태 관리를 이용해서 입력 폼, 필터, 완료 상태, 통계 정보를 화면에 바로 반영하도록 구성했습니다. 또한 사용자가 선택한 테마나 필터 같은 개인 설정은 localStorage에 저장되도록 했고, 작성 중인 입력값은 sessionStorage에 저장되도록 했습니다. 덕분에 새로고침을 하더라도 일부 화면 설정과 작성 중인 내용이 유지됩니다.

백엔드는 Node.js와 Express로 만들었습니다. 서버는 계획 목록 조회, 계획 등록, 완료 상태 변경, 삭제 기능을 API로 제공합니다. 데이터베이스는 PostgreSQL을 사용했고, 서버가 시작될 때 필요한 테이블이 없으면 자동으로 생성되도록 했습니다.

로컬 개발과 실행 환경은 Docker Compose로 구성했습니다. Docker Compose를 실행하면 프론트엔드, 백엔드, PostgreSQL, Nginx가 함께 실행됩니다. 이때 Nginx는 가장 앞단에서 요청을 받는 리버스 프록시 역할을 합니다. 사용자가 브라우저에서 접속하면 `/` 경로의 요청은 React 클라이언트로 전달하고, `/api`로 시작하는 요청은 Express 서버로 전달하도록 설정했습니다. 이렇게 구성하면 사용자는 하나의 주소로 접속하지만, 내부에서는 화면 요청과 API 요청이 분리되어 처리됩니다.

배포는 Render를 기준으로 구성했습니다. Render에서는 React 빌드 결과와 Express 서버를 하나의 웹 서비스에서 실행하고, PostgreSQL 데이터베이스도 Render에서 연결하도록 했습니다.

CI/CD 파이프라인은 GitHub Actions와 Render 자동 배포를 함께 사용했습니다. main 브랜치에 코드가 push되면 GitHub Actions가 먼저 실행되어 클라이언트 의존성 설치, React 빌드, 서버 의존성 설치, Express 서버 문법 검사를 수행합니다. 동시에 Render는 GitHub 저장소와 연결되어 있기 때문에 main 브랜치의 변경 사항을 감지해 자동으로 배포를 진행합니다. 즉, GitHub Actions로 코드가 정상적으로 빌드되는지 확인하고, Render를 통해 실제 서비스 배포가 자동으로 이루어지는 구조입니다.

정리하면 이 프로젝트는 공부계획을 기록하는 간단한 서비스이지만, 내부적으로는 프론트엔드, 백엔드, 데이터베이스, Docker, Nginx, GitHub Actions, Render 배포까지 연결한 전체 웹 서비스 구조를 경험할 수 있도록 만든 프로젝트입니다.

이상으로 공부계획 기록 사이트 발표를 마치겠습니다.
